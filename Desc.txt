Вопрос 1

Представь, что тимлид поручил тебе написать простой микросервис на Python, который будет выполнять роль ботофермы.

Что такое ботоферма?

Ботоферма — сервис, который отдает реальные креды пользователя, чтобы использовать его в E2E-тестах для проверки работоспособности функционала.

К примеру, есть веб-магазин по продаже корма и аксессуаров для животных, и есть пользователь, который может зарегистрироваться в этом магазине.

После выполнения авторизации пользователь может взаимодействовать с магазином: выбирать различные товары, их количество, их свойства, класть в корзину и так далее. Мы, как разработчики, заинтересованы в тестировании функционала "наживую", чтобы отлавливать баги, улучшать функционал, чтобы пользователь был доволен и приносил нам деньги.

Для этого нам нужен "пользователь", который мог бы авторизоваться в нашем магазине и, выполняя определенные запросы (REST-запросы в backend), эмулировал реальную работу. Ботоферма как раз реализует "выдачу" этого пользователя, которого уже будет использовать платформа для тестирования.

Что требуется сделать?

Написать RESTful-сервис на Python с использованием FastAPI в качестве веб-фреймворка, SQLAlchemy в качестве ORM, который бы:

создавал нового пользователя в ботоферме,
выдавал список всех существующих пользователей,
блокировал пользователя для его использования в рамках E2E-теста.
Что такое блокировка?

Если потребуется выдать пользователя для E2E-теста — мы должны повесить на него временную метку (locktime), когда этот пользователь был выдан, чтобы понимать, что сейчас этот пользователь занят, и его нельзя выдать для другого E2E-теста.

Если более формально:
реализовать метод create_user — создать нового пользователя в ботоферме;
реализовать метод get_users — получить список всех существующих пользователей;
реализовать метод lock_user — получить пользователя для нужд ботофермы;
реализовать метод free_users — снять блокировки со всех пользователей в системе;
Особенности lock_user

Данный метод осуществляет "блокировку" — ищет первого незанятого пользователя в системе, накладывает на него блокировку в виде временной метки, которая отображает, что пользователь занят и его можно выдать повторно только по превышении данной временной метки. После "блокировки" пользователь отдается для нужд ботофермы.

Сущности

USER:

id — UUID пользователя;
created_at — дата создания пользователя;
login — почтовый адрес пользователя;
password — пароль пользователя (должен храниться в зашифрованном виде);
project_id — UUID проекта, к которому принадлежит пользователь;
env — название окружения (prod, preprod, stage);
domain — тип пользователя (canary, regular);
locktime — временная метка (timestamp).
Инструменты:
FastAPI + uvicorn в качестве ASGI;
Pydantic версии 1 или 2;
SQLAlchemy версии 2;
PostgreSQL версии 14 и выше;
pytest.
Что будет проверяться:
сервис написан в асинхронном стиле (с использованием asyncio и asyncpg);
общая архитектура проекта (разделение валидационной и сервисной логики, следование REST, как организован код, как организована работа со схемами, как организована работа с requirements и переменными окружения);
работа сервиса (валидация входных данных, как создается пользователь, как назначается/снимается блокировка);
работа с базой (как устроена работа с сессиями, используется ли Dependency Injection, как оформлен CRUD);
стилистика кода (как названы функции, наличие doc-strings и type-hints, оформление по PEP);
покрытие проекта юнит-тестами (не менее 75%);
сервис декомпозирован в виде контейнеров и способен разворачиваться при помощи Docker-compose;
Что будет оценено дополнительно (не обязательно к выполнению):
сервис использует alembic для работы с миграциями в БД;
сервис использует механизм авторизации (например, через OAuth2);
у сервиса имеются пробы (startup, liveness, readiness);
имеются проверки кода статическими и динамическими анализаторами (ruff, mypy);
сервис способен разворачиваться в minikube при помощи helm-чартов.
Как отправить решение?

Оформите решение в любом удобном вам IDE, создайте архив в формате .tar.gz/.zip, сложите в него ваш проект и все необходимые файлы и загрузите на платформу.
